🟢 方案 A: 文件监控 (Watchdog) 推荐

原理： MQL4 脚本导出 CSV → Python watchdog 监听目录 → 自动导入 Flask

优势： 稳定可靠，无需开放网络端口，MQL4 端只需写入本地文件

⚙️ 启动步骤

1 确认 MT4 已配置并运行了 TradeExport 脚本（CSV 模式）
2 安装依赖：pip install watchdog requests
3 运行监控脚本： 
# 自动检测 MT4 目录并启动监控
python mt4_connect/mt4_watchdog.py

# 或手动指定目录和 Flask 地址
python mt4_connect/mt4_watchdog.py --dir "%APPDATA%\MetaQuotes\Terminal\*\Files" --url http://127.0.0.1:5000

# 仅扫描一次（不持续监控）
python mt4_connect/mt4_watchdog.py --scan
点击后自动扫描 MT4 目录中的 CSV 文件

🔵 方案 B: ZeroMQ 实时桥接

低延迟

原理： MT4 端运行 ZeroMQ EA → Python 订阅 ZeroMQ 消息 → 实时推送到 Flask

优势： 实时推送，低延迟，适合需要即时复盘

MT4 端额外步骤： 需要安装 mql-zmq 库并编译 EA

⚙️ 启动步骤

1 MT4 端安装 ZeroMQ 库并运行配套 EA
2 安装依赖：pip install pyzmq requests
3 运行桥接脚本： 
# 启动 ZeroMQ 桥接（默认端口 5555）
python mt4_connect/mt4_zmq_bridge.py

# 指定自定义端口
python mt4_connect/mt4_zmq_bridge.py --port 5556 --host 127.0.0.1
